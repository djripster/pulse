# Theme First, Pulse Second

## Philosophy

As Pulse evolves, it must never become a replacement for improvements that belong in the core DJs Mobiles experience.

Whenever a new Pulse feature reveals an opportunity to improve the website itself, the improvement should first be implemented within the theme.

Only after the underlying capability exists should Pulse personalize or extend it.

## Development Order

### 1. Improve the Theme

Ask:

> Can this improve DJs Mobiles for every visitor?

If the answer is yes, build it into the website first.

Examples include:

* Related article discovery
* Internal linking
* Search
* Homepage organization
* Navigation
* Content presentation

These improvements benefit every reader regardless of whether they use Pulse.

---

### 2. Personalize with Pulse

Once the website provides a strong foundation, Pulse may enhance the experience for returning readers.

Pulse should personalize capabilities rather than replace them.

Examples include:

* Prioritizing related articles based on reader interests
* Remembering favourite categories
* Continuing previous reading journeys
* Highlighting stories most relevant to the individual reader

Pulse builds upon the website instead of compensating for missing functionality.

---

## Related Content Example

### Theme

The website determines which articles are editorially related through categories, tags and content relationships.

### Pulse

Pulse looks at those related articles through the lens of the reader's history and interests.

Rather than generating an entirely different recommendation system, Pulse personalizes the existing one.

The website answers:

> "Which articles are related?"

Pulse answers:

> "Which of those articles is most meaningful for this reader?"

---

## Product Principle

Theme builds capabilities.

Pulse personalizes capabilities.

Never reverse that relationship.

The website should always become stronger first.

Pulse should always make an already great experience feel personal rather than acting as a substitute for missing functionality.

---

## Long-Term Vision

Pulse is a layer across the DJs Mobiles ecosystem.

The website, browser extension and future Android application all share the same philosophy:

* Build an excellent experience for everyone.
* Make that experience more personal over time.
* Never force accounts or personalization.
* Respect the reader by enhancing discovery rather than demanding attention.

This principle should guide future development whenever a feature could reasonably belong to both the website and Pulse.
